# 빠른 시작

먼저 OpenAPI `paths` 타입을 준비합니다. 보통 `openapi-typescript` 같은 도구로 생성하지만, 호환되는 TypeScript 타입이면 직접 작성한 타입도 사용할 수 있습니다.

## 1. 설치

`fetcher`는 함께 필요한 패키지인 `ky`와 같이 설치합니다.

```package-install
@ilokesto/fetcher ky
```

```ts lineNumbers
import { createFetcher } from '@ilokesto/fetcher/openapi';
import type { paths } from './__generated__/openapi';

const api = createFetcher<paths>({
  prefixUrl: 'https://api.example.com',
  retry: { limit: 1 },
});
```

## 타입 안전한 단축 메서드 호출하기

단축 메서드는 경로, 쿼리, 본문을 한 객체에 묶는 요청 규칙을 사용합니다.

```ts lineNumbers
const profile = await api
  .get('/users/{id}', {
    params: {
      path: { id: '42' },
      query: { include: 'profile' },
    },
  })
  .json();
```

경로 템플릿은 `ky`에 도달하기 전에 실제 값으로 채워집니다. 쿼리 매개변수는 `searchParams`로 변환되고, `.json()` 결과 타입은 OpenAPI 작업에서 선호되는 JSON 응답을 기준으로 추론됩니다.

## 타입 안전한 본문 보내기

```ts lineNumbers
const post = await api
  .post('/posts', {
    json: {
      title: 'Typed facade',
    },
  })
  .json();
```

`json`, `formData`, `formUrlEncoded`는 서로 동시에 쓸 수 없는 단축 본문 형태입니다. 세 번째 인자는 `timeout`, `signal`, `hooks`, `context`, 명시적인 본문 덮어쓰기 같은 일반 `ky` 옵션에 사용하세요.

## ky 조합 방식 유지하기

```ts lineNumbers
const authed = api.extend({
  headers: {
    Authorization: `Bearer ${token}`,
  },
});

await authed.get('/users/{id}', {
  params: { path: { id: '42' } },
});
```

`create`, `extend`, 훅, 재시도, timeout, 커스텀 `fetch`, `HTTPError` 동작은 `ky`와 호환되게 유지됩니다. `fetcher`는 OpenAPI 형태의 입력을 준비한 뒤 내부 `KyInstance`에 요청을 위임합니다.

## 예외를 원하지 않으면 safe 사용하기

```ts lineNumbers
const result = await api.safe.get('/users/{id}', {
  params: { path: { id: '42' } },
});

if (result.ok) {
  result.data;
} else {
  result.error;
  result.response;
}
```

기본 surface는 여전히 `ky`처럼 throw합니다. `safe`는 같은 request를 discriminated result로 변환합니다.
