# 문제 해결

## `prefixUrl`가 leading slash를 거부합니다

Raw ky instance가 아니라 fetcher client로 string OpenAPI path를 호출하세요. Fetcher는 알려진 ky leading-slash prefixUrl error를 slash 제거 후 retry합니다.

`extend({ prefixUrl: undefined })`로 `prefixUrl`을 제거했다면 absolute URL 또는 ky가 허용하는 path를 사용하세요.

## Path parameter가 빠졌습니다

`/users/{id}` 같은 template은 shortcut call에서는 `params.path.id`, callable call에서는 `path.id`가 필요합니다.

```ts
api.get('/users/{id}', {
  params: { path: { id: '42' } },
});
```

## Cookie parameter가 network에 나타나지 않습니다

`params.cookie`는 현재 runtime에서 type-only입니다. Cookie header가 필요하다면 `headers`로 명시하거나 browser credential behavior에 맡기세요.

## Form upload의 content type이 이상합니다

Boundary까지 직접 제어하는 경우가 아니라면 `multipart/form-data`의 `Content-Type`을 직접 설정하지 마세요. Platform이 `FormData` body에서 생성하도록 두세요.

## safe가 raw text를 반환할 줄 알았습니다

`safe`는 JSON을 읽고 성공 시 parsed `data`를 반환합니다. Text, blob, stream, custom body handling이 필요하면 기본 ky-style surface를 사용하고 response를 직접 읽으세요.
